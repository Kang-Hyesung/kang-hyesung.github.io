# 📝 Microsoft Teams & VDI 관리 핵심 요약 노트 (Updated)

## 1. 미팅 및 이벤트 유형 구분 (Meeting Types)

부서별 요구사항에 따라 적절한 미팅 유형을 선택하는 것이 핵심이다.

| 유형            | 핵심 키워드 & 특징                                             | 추천 시나리오                                          |
| ------------- | ------------------------------------------------------- | ------------------------------------------------ |
| **Webinar**   | **등록 페이지(Registration)**, 교육, 워크숍, **Whiteboard 사용 가능** | **마케팅 행사**, 외부 인원 등록 및 관리 필요 시, 인터랙티브 세션         |
| **Town hall** | **eCDN 통합**, 대규모 인원(10k~20k), **Q&A**, **단방향** 방송       | **재무/전사(All-hands) 미팅**, CEO 연설, 네트워크 부하 분산 필요 시 |
| **Meeting**   | **양방향 협업(Whiteboard)**, 소규모                             | 팀 주간 회의, 브레인스토밍, 일반적인 업무 협의                      |

> **💡 암기 팁:**
> * **Marketing 부서** + 등록 페이지 + Whiteboard = **Webinar**
> * **Finance 부서** + 1,500명 이상 + eCDN = **Town hall**
> * Webinar는 Meeting의 확장판이라 Whiteboard 사용이 가능하다.

## 2. Teams on AVD (VDI 환경 최적화)

가상 데스크톱(AVD)에서 Teams를 돌릴 때는 **레지스트리**와 **RDP 속성** 설정이 시험에 자주 나온다.

### A. 레지스트리 설정 (HKLM)

Teams에게 "너 지금 VDI 환경이야"라고 알려주거나 기능을 제어한다.

* **미디어 최적화 활성화:**
* 경로: `HKLM\SOFTWARE\Microsoft\Teams`
* 값: **`IsWVDEnvironment` = 1** (필수)


* **화면 공유 제어 (WebRTC Redirector):**
* 로컬 PC 화면 공유 허용: **`ShareClientDesktop` = 1**
* VDI 데스크톱 공유 차단: **`DisableRAILScreenSharing` = 1**
* *(주의: Disable로 시작하는 키는 1이 차단, 0이 허용 / Share로 시작하는 키는 1이 사용, 0이 미사용)*



### B. RDP 속성 (RDP Properties)

**미디어 최적화가 구성되지 않았을 때(NOT configured)**, 오디오 리디렉션 설정은 **개별 VM(Host)이 아니라 호스트 풀(Pool) 레벨**에서 한다.

* **오디오 듣기 (Playback):** **`audiomode:i:0`** (오디오를 로컬 장치에서 재생)
* **마이크 사용 (Capture):** `audiocapturemode:i:1`
* **적용 대상:** **Host Pool** (예: `Pool1`)

## 3. Teams 확장성 및 앱 통합 (Extensibility)

외부 앱(Project Management 등)을 Teams에 붙일 때 "어떤 방식"으로 붙일지 결정해야 한다.

| 기능                    | 사용 목적 (Trigger)                               | 특징                                             |
| --------------------- | --------------------------------------------- | ---------------------------------------------- |
| **Tab (탭)**           | **직접 접근(Direct Access)**, **협업(Collaborate)** | **채널 상단 고정**. 팀원들이 **동일한 화면(프로젝트 보드)**을 보며 작업. |
| **Connector**         | **알림 수신(Receive Notifications)**              | 외부 서비스(GitHub, Jira 등)의 상태 변경을 채널에 단방향으로 알림.   |
| **Message Extension** | **워크플로 생성**, 콘텐츠 검색 및 공유                      | 메시지 작성 창이나 게시물 메뉴(...)에서 바로 액션 실행.             |

## 4. 네트워크 및 정책 관리 (Network & Policy)

지사(Branch Office) 등 특정 위치에서만 대역폭을 제한하는 시나리오다.

1. **Network Roaming Policy:** 사용자의 **물리적 위치(IP 서브넷)**를 감지한다. ("어? 너 지금 Buffalo 지사에 있네?")
2. **Meeting Policy:** 실제 **대역폭 제한(Bit rate 제한)** 설정을 담고 있다.
* *설정 항목:* `Video bit rate (Kbps)` 또는 `Limit network bandwidth`


3. **작동 원리:** Roaming Policy가 위치를 파악하면 해당 위치에 할당된 Meeting Policy를 강제 적용한다.

> **💡 포인트:** "Bandwidth restriction" 문제에서 구체적인 제한 수치는 **Meeting Policy**에서, 장소 구분은 **Roaming Policy**에서 담당한다.

## 5. Teams Rooms 관리 (Devices)

회의실 장비(MTR) 설정과 라이선스 비용 최적화 문제다.

### A. 설정 파일 (Configuration)

* **파일 이름:** **`SkypeSettings.xml`**
* **용도:** 레이아웃 커스터마이징, 테마 설정, 기본값 변경 등.
* **오답 주의:** JSON 파일 아님.

### B. 라이선스 선택 (Licensing)

**비용 최소화(Minimize costs)** 조건이 있어도 기능이 우선이다.

* **Teams Rooms Basic:** 무료(25대까지). 하지만 **Audio Conferencing(전화 참여) 기능 없음.**
* **Teams Rooms Pro:** 유료. **Audio Conferencing**, 고급 관리, 화이트보드 풀 기능, **전화 기능(Phone Standard)** 포함.
* **핵심 판별법:** 지문에 "Audio Conferencing 지원 필수"라고 나오면 무조건 **Pro**가 정답.

## 6. 감사 및 거버넌스 (Audit & Governance)

Wingtip Toys 케이스 스터디 등에서 자주 묻는 거버넌스 요구사항별 해결 도구 구분이다.

| 요구 사항                  | 해결 도구 (Solution)               | 비고                              |
| ---------------------- | ------------------------------ | ------------------------------- |
| **팀 유형(Private) 설정**   | **Team Template**              | 생성 시 구조/설정 정의                   |
| **채널 내 탭 고정(Website)** | **Team Template**              | 자주 쓰는 앱/웹 미리 세팅                 |
| **팀 이름 차단/명명 규칙**      | **Entra ID Naming Policy**     | Blocklist, Prefix(GRP-), Suffix |
| **메시지 삭제/GIF 방지**      | **Messaging Policy**           | 사용자의 채팅 기능 제어                   |
| **동적 소유자/로고 할당**       | **Power Automate / Graph API** | 템플릿만으로는 동적 로직 불가                |

### 로그 추적 (Audit)

* **앱 설치/삭제/카탈로그 변경:** **Compliance Manager (Purview) > Audit Log**.
* **Entra ID 로그:** 주로 로그인, 계정 생성 등 ID 관련 이슈일 때만 정답.
* **Usage Report:** 단순히 "몇 명이 썼나" 통계만 볼 때 정답.
