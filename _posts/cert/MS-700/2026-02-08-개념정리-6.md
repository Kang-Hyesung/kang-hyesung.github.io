### 1. 보고서 & 분석 도구 (Analytics & Reports)

가장 헷갈리기 쉬운 부분임. **"무엇을(Who/What)"** 보고 싶은지에 따라 정답이 갈린다.

| 목적                                       | 정답 보고서/도구                             | 핵심 키워드                                       |
| ---------------------------------------- | ------------------------------------- | -------------------------------------------- |
| **특정 사용자**의 통화 품질/장치 드라이버 확인             | **Manage users > Meetings & Calls**   | Call Analytics, Driver version, User Profile |
| **개별 사용자**의 활동량 (비디오 시간, 임시 회의 수)        | **Teams User Activity Report**        | Ad-hoc meetings, Video time, Message count   |
| **팀(Team)** 자체의 활성/비활성 여부 (90일)          | **Teams Usage Report**                | Inactive teams, 90 days, Active users 0      |
| **비용(Cost)** 및 **분(Minute)** 확인 (오디오 회의) | **Audio Conferencing dial-out usage** | Organizer costs, Dial-out                    |
| **장치(Device)**의 PSTN 발신 분량 확인            | **PSTN & SMS usage**                  | Outbound minutes, Device usage               |
| **TCP vs UDP** 프로토콜 확인                   | **CQD > Quality Drill Down Reports**  | Transport, TCP/UDP, Detailed analysis        |

> **💡 시험 팁:**
> * **User** Activity = "사람"이 얼마나 열심히 썼나.
> * **Usage** = "팀"이나 "서비스"가 얼마나 돌아가나.
> * **Drill Down** = 겉으로 안 보이는 "기술적 세부 사항(프로토콜)" 파기.

### 2. 품질 데이터 구성 (CQD & Topology)

데이터가 보여지는 방식(IP vs 위치 이름)을 설정하는 문제다.

| 설정 항목 | 용도 및 특징 | 필수 조건 |
| --- | --- | --- |
| **Upload Building Data** | CQD에서 IP 주소를 **실제 위치(건물/층)**로 매핑 | **First step** to identify location issues |
| **Reporting Labels** | Call Analytics에서 서브넷에 **이름(Name)** 부여 | Upload .csv/.tsv, User location name |
| **Device State Rule** | 장치가 **오프라인(Offline)**될 때 실시간 알림 | Notifications & alerts, Real-time |

### 3. 정책 및 보안 (Policies & Security)

관리자가 제어하거나 감시해야 할 때 쓰는 도구들이다.

| 시나리오 | 해결 방법 | 비고 |
| --- | --- | --- |
| **팀 생성 시** 관리자 알림 받기 | **Purview > Alert Policy** | Audit log 기반, Security policy |
| **게스트 초대 오류** 원인 찾기 | **Self-help diagnostics** | MAC Help menu, "Guest user" |
| **발신 번호(Caller ID)** 숨기기/변경 | **Calling Line Identification Policy** | **Teams PowerShell**, Service Number |

### 4. 관리자 권한 (Roles - 최소 권한 원칙)

"보고서만 보게 해줘"라는 요청이 왔을 때 주는 권한이다.

1. **Usage Summary Reports Reader**: 가장 구체적이고 제한적인 정답. (보고서 **통계**만 열람 가능)
2. **Global Reader**: 위 역할이 보기에 없다면 이게 정답. (테넌트 전체 읽기 전용)
3. **Teams Communications Support Specialist**: **Call Analytics**를 통해 **개별 통화 품질** 문제 해결 시 사용. (조직 전체 보고서용 아님)

---

* **UDP**: Teams의 친구. 방화벽에서 **3478-3481** 포트를 열어야 함. (TCP로 넘어가면 품질 저하)
* **PSTN & SMS Usage**: "돈(Cost)"이나 "몇 분(Minutes)" 썼냐고 물으면 무조건 이 보고서임.
* **Purview**: "알림(Alert)"이나 "감시(Compliance)" 키워드가 나오면 Teams 관리 센터가 아니라 여기서 찾을 확률이 높음.
