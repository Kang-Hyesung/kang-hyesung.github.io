## 1. Teams Phone & Voice (음성 및 전화 시스템)

가장 디테일하게 물어보는 영역이다. **라이선스**와 **설정 위치**를 정확히 매칭해야 한다.

### Enterprise Voice (엔터프라이즈 보이스)

* **증상**: 사용자가 통화 설정에서 '전화 전달(Forwarding)'이나 '리디렉션' 메뉴가 안 보인다고 한다.
* **원인**: 해당 사용자에게 **Enterprise Voice** 기능이 비활성화(Disabled)되어 있기 때문이다.
* **해결**: PowerShell(`Set-CsPhoneNumberAssignment`) 등을 통해 활성화해야 UI가 나타난다.

### Auto Attendant (자동 응답기) 핵심 설정

* **Dial Scope (다이얼 범위)**: 특정 사용자나 그룹(예: 임원진)이 자동 응답기 검색(이름으로 전화 걸기)에서 **제외**되길 원할 때 설정한다.
* **Voice App (음성 앱)**: 자동 응답기에서 1번을 눌렀을 때 **다른 콜 큐(Call Queue)**나 **자동 응답기**로 연결하고 싶다면, 리디렉션 대상을 **Voice App**으로 선택해야 한다.

### Common Area Phone (공용 영역 전화기)

* **Hotline (핫라인)**: 수화기를 들자마자 특정 번호로 걸리게 하려면?
* 설정 위치: **Configuration Profile** (구성 프로필)
* 옵션명: **Enable hotline: On** (Advanced calling이 아님을 주의!)


* **계정 유형**: 리소스 계정이 아니라 **User Account**(사용자 계정)를 생성하고 **Shared Device** 라이선스를 할당하는 것이 정석이다.

### 라이선스 수량 산정 공식

* **사용자(User)**: 전화를 거는 사람 수만큼 **Teams Phone Standard** 필요.
* **시스템(System)**: 자동 응답기(AA) + 콜 큐(CQ)의 개수만큼 **Resource Account** 라이선스 필요. (AA와 CQ는 사람이 아니므로 Standard를 주지 않는다.)

## 2. Devices & Rooms (장치 및 회의실)

장치 관리의 핵심은 **'보안'**과 **'원격 제어'**다.

### Teams Rooms 라이선스: Pro vs Basic

* **Teams Rooms Pro**: **Intelligent Speaker**(목소리 식별), **Intune** 관리, 고급 AI 기능이 필요하면 무조건 Pro다.
* **Teams Rooms Basic**: 무료지만 기능 제한이 많다(최대 25대).

### 원격 프로비저닝 (Remote Provisioning)

현장 작업자에게 **비밀번호를 알려주지 않고** 기기를 로그인시키는 절차다. 순서를 외워두는 게 좋다.

1. 관리자: **MAC 주소** 업로드 (또는 인증 코드 생성 준비)
2. 관리자: **Verification Code**(인증 코드) 생성
3. 현장 작업자: 기기에 **코드 입력**
4. 관리자: 관리 센터에서 **로그인 승인**

## 3. Meetings & Collaboration (모임 및 협업)

**보안 설정(E2EE)**과 **외부 접속(B2B)**의 차이를 묻는 문제가 단골이다.

### 미팅 보안과 기능의 Trade-off

* **종단 간 암호화(E2EE) 켜짐**: 보안은 최강이지만 서버가 데이터를 못 읽는다.
* **사용 불가 기능**: **Microsoft 365 Copilot**, 녹화(Recording), 실시간 자막(Live Caption/Transcription).

### External Access vs Guest Access (가장 헷갈리는 개념)

* **External Access (Federation)**: 그냥 채팅하고 전화만 할 때. (Teams 관리 센터에서 설정)
* **Cross-tenant Access (B2B Collaboration)**: 상대방 **팀 채널(Team Channel)**에 들어와서 협업해야 할 때. (Microsoft Entra ID에서 설정)
* Wingtip 사용자가 Fabrikam 팀에 못 들어가게 막으려면? -> **Entra ID**의 **Outbound settings**에서 차단.



### Webinar vs Town hall

* **Webinar**: **등록(Registration)** 페이지가 필요하고 Q&A가 중요한 행사.
* **Copilot 라이선스**: "이벤트 중/후에 Copilot을 쓴다"면 주최자뿐만 아니라 **참여하는 모든 사용자**에게 라이선스가 있어야 한다.

## 4. Troubleshooting & Network (문제 해결 및 네트워크)

어떤 상황에 **어떤 도구**를 써야 하는지 매칭하는 싸움이다.

### 도구 매칭 (Tools Matching)

| 목적                         | 정답 도구                            |
| -------------------------- | -------------------------------- |
| 배포 전 구성 점검 / 체크리스트         | **Teams Advisor**                |
| IP/포트가 열려 있는지 연결 확인        | **Remote Connectivity Analyzer** |
| 대역폭(Bandwidth) 계산 / 사이트 설계 | **Network Planner**              |
| 통화 품질 사후 분석 / 건물별 데이터      | **Call Quality Dashboard (CQD)** |
| 3rd Party 앱 로딩 실패 디버깅      | **Browser Dev Tools (F12)**      |

### CQD "Outside" 문제 해결

* 내부망인데 Outside로 뜬다면? CQD가 우리 회사 IP 대역을 모르는 것이다.
* 해결: **Tenant Data Upload** (서브넷 정보를 담은 CSV 파일 업로드).

### 미디어 로깅 (Media Logging)

* 오디오 끊김 문제 등으로 상세 디버깅이 필요할 때 사용자가 로그를 수집해도 내용이 없을 수 있다.
* 해결: `Grant-CsTeamsMediaLoggingPolicy -Identity User1 -PolicyName Enabled` 명령어로 정책을 켜줘야 한다.

---

### 마지막 팁: 최소 권한 (Least Privilege)

* 네트워크 계획이나 통화 품질 관리만 하면 된다면 **Teams Administrator**(전역 관리)를 주지 마라.
* 정답: **Teams Communications Administrator** (통화/미팅/네트워크 전담).
